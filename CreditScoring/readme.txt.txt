CreditScoring

Сервис-заглушка скоринга заявок на кредит.
REST микросервис, паттерн репозиторий.

Краткое описание работы:
 ScoringController по апи api/scoring/evaluate получает заявку на скоринг,
CreditScoringManager преобразует входящий запрос в модель внутреннего слоя
и обращается к репозиторию-заглушке для её обработки.
 Решение о скоринге принимается в самом же репозитории и при помощи SendScoringResult 
передаётся сервису обработки заявок на кредит.
 Примечание: постарался сделать скоринг-заглушку таким образом чтобы для
превращения заглушки в полноценное приложение понадобилось минимум правок существующего кода. 
По идее для этого достаточно переписать репозиторий.

Appsettings:
CreditEvaluate - скоринг заявок на кредит
 apiUri - апи для отправки результатов скоринга
 WaitingTimeMs - время ожидания для отправки результатов скоринга

Структура:

Controllers - контроллеры для обработки входящих http запросов

Data - управление данными
 Repository - манипуляции с данными, CRUD
 SendResult - передача результатов скоринга по http

Domain - middleware
 Managers - преобразование данных для слоёв приложения

Infrastructure - расширения для приложения
 Mapper - профайл для библиотеки automapper
 StartupExtensions - расширения для класса startup

Models - объекты данных
 Entities - модели для внутреннего использования
 EvaluateRequest - модели входящего запроса на скоринг
 ScoringResultRequest - модели с результата скоринга
